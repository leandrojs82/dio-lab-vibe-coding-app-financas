
# Guia de Contribuição

Obrigado por considerar contribuir com este projeto.
Toda contribuição é bem-vinda, independentemente do seu nível de experiência.

Este projeto tem como princípio central a **simplicidade**, a **inclusão** e o **Design Universal**. Ao contribuir, leve esses valores em consideração.

---

## Como Contribuir

Você pode contribuir de várias formas:

* Relatando problemas (issues)
* Sugerindo melhorias
* Melhorando a documentação
* Propondo melhorias de acessibilidade e usabilidade
* Contribuindo com código
* Revisando textos e linguagem para torná-los mais claros e acessíveis

---

## Antes de Começar

Antes de abrir uma issue ou pull request:

1. Verifique se já existe uma issue relacionada
2. Leia o README para entender o propósito do projeto
3. Mantenha as contribuições alinhadas ao foco do MVP

---

## Abrindo Issues

Use issues para:

* Relatar bugs
* Sugerir novas funcionalidades
* Propor melhorias de acessibilidade
* Discutir ideias e melhorias

Ao abrir uma issue, sempre que possível, inclua:

* Descrição clara do problema ou sugestão
* Passos para reproduzir (se for um bug)
* Comportamento esperado
* Contexto adicional relevante

Use linguagem clara, simples e objetiva.

---

## Enviando Pull Requests

### Passos Básicos

1. Faça um fork do repositório
2. Crie uma branch a partir da branch principal
3. Faça suas alterações
4. Garanta que o código esteja funcionando
5. Envie um pull request

---

### Boas Práticas

* Mantenha mudanças pequenas e focadas
* Use nomes claros para commits e branches
* Evite adicionar complexidade desnecessária
* Priorize legibilidade e simplicidade
* Escreva código e textos acessíveis

---

## Diretrizes de Design Universal

Ao contribuir, verifique se sua proposta:

* Funciona para o maior número possível de pessoas
* Evita dependência excessiva de elementos visuais
* Utiliza linguagem simples e compreensível
* Minimiza esforço cognitivo
* É tolerante a erros do usuário

Acessibilidade não é um diferencial, é um requisito.

---

## Estilo de Código

Enquanto o projeto está em evolução:

* Prefira código simples e legível
* Comente apenas quando necessário para clareza
* Evite jargões técnicos desnecessários
* Priorize soluções fáceis de manter

Padrões mais específicos poderão ser definidos futuramente.

---

## Comunicação e Respeito

Este projeto valoriza um ambiente colaborativo, respeitoso e inclusivo.

* Seja gentil e construtivo
* Respeite diferentes níveis de experiência
* Aceite feedback de forma aberta
* Evite linguagem ofensiva ou excludente

---

## Revisão e Aprovação

Todos os pull requests serão revisados.

Durante a revisão, podem ser solicitados ajustes para:

* Melhorar clareza
* Aumentar acessibilidade
* Simplificar soluções
* Manter alinhamento com o MVP

---

## Dúvidas

Se tiver dúvidas sobre como contribuir:

* Abra uma issue com sua pergunta
* Explique claramente o que você pretende fazer

A comunidade e os mantenedores ajudarão sempre que possível.

---

Obrigado por contribuir.
Este projeto existe para tornar o controle financeiro mais simples, humano e acessível para todos.
